* Local> Rebuild and push to deploy:

git push origin main

docker build --platform linux/amd64 -t nunoloureiro/vulnapps:latest .
or
docker build --platform linux/amd64 --no-cache -t nunoloureiro/vulnapps:latest .

docker push nunoloureiro/vulnapps:latest

* EC2> Update to latest version:

sudo docker stop vulnapps; sudo docker rm vulnapps
sudo docker pull nunoloureiro/vulnapps:latest
source ~/.env.vulnapps
sudo docker run -d --name vulnapps --restart unless-stopped -p 127.0.0.1:8001:8000 -v vulnapps-data:/data -e SECRET_KEY="$VULN_SECRET" -e DATABASE_PATH=/data/vulnapps.db -e TOKEN_EXPIRY_HOURS=24 nunoloureiro/vulnapps:latest

sudo docker image prune -f



* EC2> First time setup:

bash aws/setup-ec2.sh

# Install nginx host config (update server_name in the file first):
sudo cp docker/nginx-host-vulnapps.conf /etc/nginx/sites-available/vulnapps
sudo ln -sf /etc/nginx/sites-available/vulnapps /etc/nginx/sites-enabled/vulnapps
sudo nginx -t && sudo systemctl reload nginx

# Optional: setup SSL with certbot
sudo certbot --nginx -d vulnapps.net




* Notes:

- Port 8001 on host (8080 is TaintedPort)
- Data persists in vulnapps-data volume across container rebuilds
- First registered user becomes admin
- Update server_name in docker/nginx-host-vulnapps.conf to your domain
